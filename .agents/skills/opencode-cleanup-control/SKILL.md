# OpenCode Cleanup Control

## Purpose

Use only for cleanup planning or cleanup execution.

Do not load for ordinary repo work.

Codex MUST already have read `opencode-mcp-windows-control`.

## Model

For cleanup tasks through this skill, use the explicit default model:

- providerID: `deepseek`
- modelID: `deepseek-v4-flash`

Do not call `opencode_provider_models` or switch models unless that exact
provider/model is rejected, unavailable, or the user explicitly requests a
different model.

## Mandatory cleanup loop

Cleanup always uses:

```text
OpenCode proposes cleanup plan -> Codex selects approved paths -> OpenCode deletes/archives only selected paths.
```

OpenCode MUST NOT decide on its own what to delete/archive.
Codex MUST NOT delete/archive repo paths locally.

## Cleanup plan

OpenCode cleanup plan must include:

- candidate path;
- category;
- evidence;
- why it may be safe;
- risk;
- recommended action: keep / archive / delete / ignore;
- whether tracked or untracked;
- whether generated by current task or pre-existing.

## Safe candidates

Possible candidates:

- temporary files;
- generated logs;
- old smoke reports;
- obsolete local artifacts;
- duplicate reports;
- cache-like project artifacts;
- gitignore candidates.

## Dangerous zones

Never delete/archive without separate explicit approval:

- `.git`;
- source files;
- config files;
- launch scripts;
- docs that may be user-facing;
- secrets;
- `.env`;
- lock files;
- database files;
- unknown files;
- files outside workspace.

## Execution

After Codex selects approved paths, OpenCode may delete/archive only those
exact paths.

If a selected path differs from the plan, stop.

If uncertain, stop and return `NEEDS_CODEX_DECISION`.
