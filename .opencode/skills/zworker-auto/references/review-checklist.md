# Чек-лист приёмки Zworker-результата

## A. Архив

- [ ] Имя ZIP точно соответствует контракту.
- [ ] Архив открывается без ошибок.
- [ ] `answer.md` находится в корне, не во вложенной папке.
- [ ] Нет абсолютных путей.
- [ ] Нет `..` и path traversal.
- [ ] Нет симлинков, ведущих за пределы целевого каталога.
- [ ] Нет файлов вне `allowed_paths`.
- [ ] Нет файлов внутри `forbidden_paths`.
- [ ] Нет `payload/`.
- [ ] Нет `manifest.json` или checksums, если они не были явно запрошены.

## B. `answer.md`

- [ ] Написан на требуемом языке.
- [ ] Описывает фактически выполненную работу.
- [ ] Перечисляет все дополнительные файлы.
- [ ] Объясняет, куда положить файлы и как использовать.
- [ ] Даёт проверяемые шаги верификации.
- [ ] Отдельно перечисляет непроверенные пункты.
- [ ] Не заявляет вымышленные результаты тестов, git или runtime.
- [ ] Указывает версии Zworker docs, если они объявлены источниками.

## C. Содержимое

- [ ] Возвращены прямые целевые файлы, если именно они требовались.
- [ ] Нет helper patcher вместо требуемой реализации.
- [ ] Нет псевдокода вместо рабочего файла.
- [ ] Существующая архитектура сохранена.
- [ ] Изменения ограничены scope задачи.
- [ ] Использованы фактически прочитанные исходники.
- [ ] Экспорты, API и существующая функциональность не удалены без требования.
- [ ] Добавленные зависимости разрешены задачей.
- [ ] Скрипты просмотрены до выполнения.
- [ ] Нет секретов, ключей или подозрительной эксфильтрации.

## D. Автоматизационный блок

- [ ] `request_id` совпадает с запросом.
- [ ] `zip_filename` совпадает с реальным файлом.
- [ ] `zip_kind: zworker_result`.
- [ ] `required_root_file: answer.md`.
- [ ] После `ZWORKER_ZIP_READY` нет текста.

## E. Решение

### ACCEPT

Использовать только если обязательные требования выполнены. Отдельно перечислить `Needs local verification`.

### REVISION REQUIRED

Составить `-verN` с точными дефектами:

- файл и участок проблемы;
- ожидаемое поведение;
- требуемая форма возвращаемых файлов;
- что нельзя повторять;
- какие опубликованные исходники являются источником истины.

### CLARIFICATION REQUIRED

Использовать, когда отсутствует точный файл, команда, формат данных или решение пользователя, без которого нельзя корректно продолжить.

### REJECT

Использовать при небезопасном архиве, выходе за scope, подмене задачи или результате, который невозможно исправить точечной ревизией.
